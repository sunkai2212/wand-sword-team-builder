# 技能图标圆环居中设计

## 目标

修正技能图标在方形素材中的视觉偏移。所有技能以圆形外框为定位基准，让圆心、圆环直径和四周留白保持一致；重点修正四至七转，同时复核一至三转，避免切换转数时出现大小或位置跳动。

## 视觉标准

- 输出尺寸继续保持 `128 × 128`，不改变网页布局、技能顺序或文件名。
- 圆形外框的圆心位于方块中心 `(64, 64)`。
- 目标圆环外径为约 `112 px`，即上下左右各保留约 `8 px` 的基础留白；允许因原图边缘和抗锯齿产生最多 `2 px` 的视觉误差。
- 居中判断只参考技能圆形外框。圆外尖角、光效、装饰和底部文字不参与圆心计算，必要时允许被裁掉。
- 不拉伸圆环，不改变宽高比，不使用网页 CSS 为不同转数单独偏移。

## 处理方案

采用按源图分组的人工校准，不增加容易误判的自动圆环识别逻辑：

1. 一至六转按每张职业/转数技能总图校准。同一排同规格图标共用圆心间距和圆环直径，只对主动、被动两排存在的真实差异分别处理。
2. 七转是独立截图，逐张确定圆形外框中心并重新裁切，移除当前统一 `240 × 240` 裁切带入的底部文字与偏移。
3. 更新 `data/source-assets.json` 中的裁切框后，通过现有 `npm run assets:build` 重建技能 WebP，不修改源图。
4. 用带中心十字线和统一圆环参考线的全量联系表进行视觉复核；优先检查四至七转，再抽查一至三转。

## 验收

- 全部 380 个技能图标仍可从仓库源图重建，输出路径和目录记录不变。
- 四个职业的一至七转图标在联系表中圆心一致，圆环大小无明显跳动。
- 七转不再显示底部文字，圆形外框完整且居中。
- `npm run assets:build`、`npm run data:check`、单元测试、生产构建和浏览器测试全部通过。
- 网页技能选择器、已选技能槽和导出 PNG 使用同一批修正后的图标，无额外显示分支。

## 非目标

- 不重绘技能图标，不修改颜色、清晰度或技能内容。
- 不修改宠物和职业形象。
- 不添加技能名称、说明或数值。
- 不借此调整页面其他间距、按钮或版式。
